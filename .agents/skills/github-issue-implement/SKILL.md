# GitHub Issue Implement Skill

## Purpose

ユーザーが選択した Issue を実装し、必要な検証を行って変更を完了させる。

## When To Use

- ユーザーが Issue 番号を指定したとき
- 既に対象 Issue が確定しているとき

## Steps

1. GitHub MCP で Issue の受け入れ条件・関連情報を確認する。
2. ローカルコードを調査し、最小差分の実装方針を決める。
3. 実装し、必要なテストを追加・更新する。
4. 可能な範囲でテスト/静的解析を実行する。
5. 変更内容・検証結果・未実施項目をユーザーに報告する。

## Output Format

- 何を変更したかを先に 1〜2 行で要約
- 変更ファイルごとの理由
- 実行コマンド結果（成功/失敗）
- 次アクション候補（必要時のみ）

## Guardrails

- issue 要件が曖昧なら実装前に確認する
- 既存の無関係な差分は戻さない
- 破壊的変更は理由と影響範囲を明記する
